# ARCHITECTURE — Caelum

The durable description of the system's shape, its non-negotiable invariants, and the log of every architectural decision. Onboarding drafted this from the `_intake/` migration brief (gitignored local planning input; see [`_intake/README.md`](../_intake/README.md)); it's updated whenever a `D-NN` decision is made. Keep it honest and current — `plan_work` and `review_against_library` both lean on it.

## 1. Shape

Caelum is a layered Angular component library. **Thesis:** *stay inside the Angular org's own stack — Angular Material + CDK + Angular Aria, all Google-maintained and MIT (US-origin) — and build the gaps on those primitives, isolating the few genuine third-party needs behind app-owned adapters.* This sidesteps the provenance-audit problem for the majority of components (brief §0, §1).

Layers, top to bottom:

```
consuming app  ─►  <cae-*> public component API     (PrimeNG-parity ergonomics, app swaps p-* → cae-*)
                          │
        ┌─────────────────┼───────────────────────────┐
        ▼                 ▼                            ▼
  composition       built primitives             adapter layer
  (Material+CDK     (CDK Overlay/A11y/DragDrop/   (the ONLY files importing an
   assembled)        Scrolling/Menu/Listbox +      admitted 3p lib — grid/chart/editor)
        │             Angular Aria headless)              │
        └─────────────────┬───────────────────────────────┘
                          ▼
                  theming token bridge   (one CSS-custom-property token set:
                  the single source of design truth; Material's theming API AND
                  every built component read the same variables)
```

- **Public component API** — Caelum's exported components/directives, shaped for PrimeNG-parity so a consuming team replaces `p-*` selectors with `cae-*` (or Material directly) with minimal call-site churn.
- **Composition layer** — components assembled from existing Material + CDK pieces (e.g. ConfirmDialog = `MatDialog` + wrapper; MultiSelect = `MatSelect[multiple]` + filter/chip-summary).
- **Built primitives** — components Material doesn't ship, built on **CDK** (Overlay, A11y/FocusTrap/LiveAnnouncer, DragDrop, Scrolling/VirtualScroll, Menu, Listbox, Stepper) and **Angular Aria** headless ARIA primitives (stable in Angular 22).
- **Adapter layer** — the *only* code that imports an admitted third-party UI library; exposes a neutral, app-owned interface (brief §2.1). Three adapters: data grid, charts, rich-text editor.
- **Theming token bridge** — one CSS-custom-property token set (color, surface, spacing, radius, typography, focus ring) consumed by Material's theming API *and* by every built component (brief §2.2).

The two load-bearing patterns (adapter isolation, token bridge) are the spine; see brief §2 and `D-03`/`D-04` below.

## 2. Invariants (the non-negotiables)

The properties that must hold no matter what. `definition_of_done` gates on them; every change is checked against them.

- **Provenance purity** — every runtime dependency *and its transitive tree* is free/permissive-licensed (absolute, no paid) and US-origin — **as refined by `D-11`: the origin axis is scoped to shipped-runtime reachability, with an allied-nation (UK/DE/…) fallback and Angular's own closure accepted wholesale; the strict "no non-US in the full installed tree" reading of D-05/D-10 is superseded.** A violation is a hard-block, not a soft-flag — mechanically enforced by `scripts/check_provenance.py` against `provenance/allowlist.json` in CI (§3.5). (brief §0; `D-05` → `D-11`)
- **Adapter isolation** — no app/component code imports an admitted third-party UI library directly; only that library's single adapter file does. Mechanically enforced by ESLint `no-restricted-imports`. (brief §2.1, §8; `D-03`)
- **Token-only theming** — built components read design values only from the token bridge (CSS custom properties); no hardcoded colors/spacing/radii/typography. (brief §2.2; `D-04`)
- **A11y parity** — every component meets or exceeds PrimeNG's keyboard nav, focus order, ARIA roles, and screen-reader labels. Direct/Material components inherit it; **built** components are explicitly axe- + keyboard-tested. (brief §6)
- **Evidence-gated done** — a component is "done" only with passing parity scenarios **plus** adversarial sign-off, never because it renders. Tracked in the capability ledger. (brief §7)
- **Angular lockstep** — track Angular / Material / CDK majors in lockstep (zero version lag is the entire reason for the stack choice); no pinning the library a major behind. (brief §0)
- **Zoneless change detection** — every component ships `ChangeDetectionStrategy.OnPush` with signal-driven inputs/state and **no zone-coupled `NgZone` / `zone.js` APIs**; the library runs zoneless-by-default and must stay runnable without `zone.js`. (Book 01 §3.2; `D-12`)

## 3. Subsystems

One subsection per major component cluster; filled in as the system grows. The book section(s) behind each land here once `build_library` runs.

- **3.1 Theming token bridge** — *built (M0-3, #3).* Two-tier `--cae-*` token set (primitive palette → semantic role tokens; color/surface/space/shape/elevation/type/focus/motion) in `projects/caelum/styles/_tokens.scss`, and the bridge in `_theme.scss`: `mat.theme()` stands Material up (theme-type defaults to `color-scheme`), then `mat.theme-overrides()` binds `--mat-sys-*` ← `var(--cae-*)` so Material controls and future `cae-*` components resolve the same values. Light/dark via `light-dark()` + `color-scheme` (auto), with `:root[data-theme]` to force an arm; components stay unaware (Book 04 §3.1–§3.3, §3.6). Shipped as raw SCSS package assets (`ng-package.json` → `assets`) so a consumer applies it with `@use 'caelum/styles/theme'`; **Forge is the first consumer** (`projects/forge/src/styles.scss`) and demonstrates it end-to-end with a light/dark toggle. **Density parity (R4) — built (#411, M4 pulled forward):** a single global knob, `:root[data-density='compact']`, re-declares the `--cae-space-*` step scale (built `cae-*` tighten) *and* re-emits Material's density at `-2` under the same selector (Material controls tighten in lockstep) — runtime-switchable, mirroring the `[data-theme]` mechanism; absence of the attribute = the comfortable `density: 0` baseline. **Adoption:** set `data-density="compact"` on the root (Forge has a toggle); no Sass param needed (a compile-time baked baseline is a follow-on). **A11y floor:** Material clamps density to each component's minimum and a density-invariant `--cae-target-min: 24px` token holds the WCAG 2.5.8 (AA) target-size floor — compact trades away the comfortable 48px touch overlay (`touch-target-display: none`) but never drops below 24×24 CSS px (Book 04 §3.4). The density-API *shape* is **D-19** (global runtime `[data-density]`, ratified #412); visual-regression across density × scheme stays M4 (#240).
- **3.2 Built component families** — *built (M0, ongoing).* Forms/inputs, buttons, panel/layout, overlay/menu, data-display, feedback/messages, media, misc directives (brief §3 migration map; the live slice index is [`ROADMAP.md`](ROADMAP.md) M0). Every `cae-*` follows one reusable **Direct-component recipe** — the CVA seam (`CaeFormFieldControlBase` vs controlled-CVA), the non-form-field `ariaDescribedby` a11y hook, cae-button directive forwarding, content-projection/DI coordination, the D-15 service-passthrough family, and the ng-packagr/bundle gotchas — captured once in [`docs/PATTERNS.md`](PATTERNS.md) (the single home for the pattern, so it isn't re-narrated per slice).
- **3.3 The three adapters** — grid, charts, editor: each a neutral interface + a single adapter file (brief §2.1, §4).
- **3.4 Parity & a11y verification harness** — the capability ledger (`untouched → mapped → implemented → parity-verified → adversarial-passed`), pre-committed scenario specs, axe + keyboard + visual-regression (brief §6, §7).
- **3.5 Provenance / compliance gate** — *built (M0-4, #4).* The manual M0-2 scan is now automated as CI/preflight gates against a curated, checked-in allowlist (`provenance/allowlist.json`, the D-11 source of truth): (a) **dependency-provenance gate** (`scripts/check_provenance.py`) — pure-Python, node-free, reconciles the resolved tree (`package-lock.json`) against the allowlist with the runtime-vs-dev split (Book 03 §3.3/§3.4): runtime deps permissive + US/allied + vetted (a new runtime dep absent from the allowlist FAILS — the forward ratchet), dev tree license-clean with weak-copyleft admitted only via a ledgered `devCopyleftExceptions` row (the axe-core precedent, Book 16 §3.2); it runs in CI's node-free **`static gates`** job so it is a real blocking check today; (b) **adapter-isolation ESLint** (`eslint.config.js` `@typescript-eslint/no-restricted-imports`, D-03/R6, Book 12 §3.3) — each admitted engine fenced to one explicit `*.adapter.ts`; (c) **two size gates** (Book 18) — the Forge-app `angular.json` budgets (§3.1) and the bespoke shipped-library per-entry-point gate `scripts/check-lib-size.mjs` (§3.2, zero-dep gzip, since `angular.json` budgets don't measure ng-packagr output); (d) the **US-origin attestation emitter** (`scripts/emit-us-origin-attestation.mjs`, Book 19 §3.5/§4(e)) — writes `us-origin.attestation.json` into the tarball, orthogonal to npm/SLSA build provenance (Book 19 §2.2). Gates (b)–(d) run in preflight now; their CI-matrix wiring + required-check promotion is #6. See `provenance/README.md`. (brief §0, §4; D-05/D-10/D-11)

## 4. Risks

From the brief's pre-loaded scars (§8) plus this repo's standalone-library constraint.

| # | Risk | Mitigation |
|---|------|------------|
| R1 | `MatTable` at scale — fine for hundreds of rows, not tens of thousands | Decide grid-vs-`MatTable` **by row count, per screen**; document the virtual-scroll threshold; the grid adapter covers the heavy case |
| R2 | Calendar time-of-day gap (`MatDatepicker` is date/range only) | Build the time-of-day picker explicitly; tracked as a slice, not assumed away |
| R3 | InputNumber / InputMask locale parity (grouping, currency, paste, negatives) | Test against locale/currency/paste/negative cases; don't ship "looks the same" |
| R4 | Theming parity is not free — Material ≠ PrimeNG out of the box | Budget explicit token work; parity scenarios snapshot under light **and** dark |
| R5 | Foreign transitive deps — a US top-level package can pull a non-US dependency | Provenance check on the dependency **tree**, not the package (scoped to shipped-runtime reachability per `D-11`); automated CI gate `check_provenance.py` (§3.5) — a new unvetted runtime dep hard-fails |
| R6 | Adapter erosion — first direct import "just this once" kills isolation | ESLint `no-restricted-imports` scoping each lib to its adapter + a review lens |
| R7 | **No access to the consuming app** — can't rank work by real `p-*` usage | Build the full parity surface ranked by *general* PrimeNG frequency; ship an adoption/usage-mapping guide so the team maps their own usage on adoption |
